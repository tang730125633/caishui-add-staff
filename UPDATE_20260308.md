# 重要更新：部门ID获取方式 (2026-03-08)

## ⚠️ 之前的问题

很多用户（包括我）在使用 API 方式时遇到了 `403 无操作权限` 错误，原因是：

1. ❌ **错误的部门ID**：使用了 `departmentIds: [1]` 这样的默认ID
2. ❌ **缺少 companyId**：API 调用时遗漏了关键参数
3. ❌ **没有获取真实部门ID**：应该从页面获取系统分配的真实部门ID

## ✅ 正确的解决方案

### 获取真实部门ID

```python
# 方法1: 使用 get_department_map.py 脚本
python get_department_map.py

# 输出示例:
# - 凯旋创智测试集团: 9147
# - 测试门店1: 9151
# - 测试门店2: 9152
# - 测试门店3: 9153
```

### 正确的 API 调用

```python
import requests

headers = {
    "x-token": "你的Token",
    "Content-Type": "application/json"
}

# ✅ 正确的参数
payload = {
    "nickName": "员工姓名",
    "mobile": "13800138000",
    "departmentIds": [9151],  # 真实的部门ID！
    "companyId": 7792         # 必须包含！
}

response = requests.post(
    "https://cst.uf-tree.com/api/member/userInfo/add",
    headers=headers,
    json=payload
)
```
